# 图像搜索

这是一个高级动作，可以扫描整个屏幕，找到目标图像的位置，并准确返回该图像中心的坐标 (X, Y)。

* 功能：当网站或应用程序没有 HTML 标签（如游戏、旧应用程序或画布图形）时，这是一个“救命”解决方案。您可以使用它来确定坐标，从而要求工具准确地单击该位置。

🎥 查看更多教程视频：[在这里](https://youtu.be/xWu0g35YTGo)。

> ⚠️ 关于缩放比例（Scale）的特别注意事项：您在拍摄样本图像时，屏幕的缩放百分比和浏览器的分辨率必须与实际运行脚本时完全相同。如果您在拍摄样本图像时将屏幕显示设置为 100%，但在运行配置时系统自动以 80% 或 125% 的缩放打开，则图像扫描算法将无法识别目标的像素结构，从而导致无法找到坐标。

#### 实际示例：在屏幕上找到移动物体的位置以单击

假设您正在一个奖励游戏网站上运行脚本，或者在一个按钮位置实时不断变化的应用程序中。您需要找出该按钮当前的位置，以便控制鼠标单击：

* 配置方法：您拍摄该按钮的样本图像作为搜索目标。
* 结果：在执行时，系统将快速扫描屏幕。如果找到按钮，它将立即计算出像素坐标（例如：`X = 550`，`Y = 420`），并将这两个值存储到系统输出变量中。随后，您可以调用 Mouse move 或 Mouse click 动作，将这两个坐标变量 `X` 和 `Y` 直接传入，以便鼠标自动找到并准确单击目标。

> 💡 _详细参数，如样本图像区域（Image）、颜色偏差（Threshold）、真实颜色（True color）和输出变量（Output）将在结合三种动作：Wait to image、Image exists 和 Image search 的部分中深入分析。_
