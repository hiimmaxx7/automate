# 文件存在

此操作帮助工具检查计算机上是否已经存在一个文件。返回结果非常简单：True（如果找到文件）或 False（如果未找到）。

* 功能：与 `If` 语句结合使用，以更智能地处理，避免工具在您要求它打开一个实际上不存在的文件时出现错误或突然停止。

🎥 查看更多教程视频：[在这里](https://youtu.be/EH7olDLAb9c)。

#### 配置参数：

* 文件路径：指向需要检查的文件的绝对路径（例如：`D:\data\accounts.txt`）。
* 输出变量：保存检查结果的变量名称（`True` 或 `False`）。

#### 实际示例：在运行脚本之前检查配置文件

假设您编写了一个自动填写信息的脚本，并要求脚本从文件 `D:\GPM\config.json` 中读取数据。如果用户忘记创建此文件或输入错误的路径，脚本在读取文件的命令处将会立即出现系统错误。

为了安全处理，您可以将脚本配置如下：

* 步骤 1：调用文件存在操作，填写路径 `D:\GPM\config.json` 并将输出变量命名为 `$isFileReady`。
* 步骤 2：使用 If 语句进行检查：`如果 $isFileReady = False`。
* 步骤 3：在 If 语句内部，调用 Throw 操作，内容为：`"错误：您尚未在 D 盘创建 config.json 文件，请检查！"`。

结果：脚本将自动进行检查。如果文件尚不存在，它将主动停止流程并发出红色警告，以便您知道，而不是继续运行然后导致脚本崩溃。
